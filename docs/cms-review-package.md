# CMS Review Package

The CMS engine can export a draft-vs-published review package from the `Releases` module.

## Purpose

Use this package when a reviewer, product owner, or backend workflow needs one JSON artifact that summarizes:

- what changed between the current draft and the latest published snapshot
- locale coverage for `en` and `pt-BR`
- the current release candidate checklist state

This package is **review-only**. It is not intended to be imported back into the CMS.

## Payload contents

The exported payload contains:

- `profile`
  - tenant identity used for the export
- `review.comparedRelease`
  - published release id, name, environment and publish timestamp
- `review.summary`
  - aggregate counters for changed pages, sections and blocks
  - locale coverage summary
  - checklist status counters
- `review.diff`
  - full draft-vs-published diff structure
- `review.localeCoverage`
  - full locale coverage matrix
- `review.checklist`
  - full release candidate checklist snapshot

## Typical use cases

- attach the package to an approval workflow
- persist review evidence outside the browser session
- feed external QA or product review tooling
- compare release readiness without giving direct CMS access

## Integration note

If a backend team wants to persist this export, treat it as an immutable review artifact generated by the CMS engine. Do not use it as the source of truth for content persistence.
